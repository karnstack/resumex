# README revamp + hero image

The current `README.md` references the old markdown layout, the dropped CodeMirror editor, and claims "two flavors out of the box" when there are now 100+ templates. This rewrites it to match the live product (TSX-component resumes, `/start` bootstrap, 100+ templates, no editor) and adds a generated hero image at the top.

## Scope

- Rewrite `README.md` top-to-bottom. Structure mirrors the karnstack `/dashboard/resumex` landing page so the brand voice stays consistent across the two surfaces.
- Add `docs/hero.png`, a 1280×640 grid of 8 template thumbnails, generated by a one-off script.
- Add `scripts/build-hero.mjs` that uses Playwright to capture each template from the live deployment, then sharp to composite them into a single PNG.
- Add `playwright` and `sharp` as `devDependencies`.
- Add `pnpm hero` to `package.json` scripts. Do NOT add it to `prepare` or `pretypecheck`. running a headless browser on every install is too slow.

Out of scope:

- Screenshots gallery / `docs/gallery.md`. Already covered by `resumex.karnstack.com/templates`.
- Animated GIFs or screencasts.
- Logo or wordmark redesign.
- Touching `docs/CONVENTIONS.md` or `docs/TEMPLATE_GUIDE.md`.

## Hero image

`scripts/build-hero.mjs`:

1. Launches headless Chromium via Playwright.
2. For each of 8 hand-picked template IDs, opens `${BASE}/preview/<id>?embed=true` at viewport 793×793 (top square of A4 at 96dpi). Waits for `networkidle` plus a short fonts-loaded pause.
3. Screenshots each as a 793×793 PNG buffer.
4. Uses `sharp` to resize each to 320×320 (`fit: 'cover', position: 'top'`), composites all 8 into a 1280×640 grid (4 columns × 2 rows), writes `docs/hero.png` with `compressionLevel: 9`.

Templates for the grid, ordered for visual rhythm:

```
row 1: swiss · emerald-twocol · editorial-vogue · terminal
row 2: brutalist-redbar · magazine-issue · architect-blueprint · banker-monogram
```

Same six the karnstack landing page features, plus `magazine-issue` and `banker-monogram` for two more aesthetic flavors.

`BASE` defaults to `https://resumex.karnstack.com` (so the script is self-contained and doesn't require a local dev server). Override with `RESUMEX_HERO_BASE=http://localhost:5173 pnpm hero` when iterating on a template locally before deploying.

The output PNG is committed to git. Regeneration is a manual step (`pnpm hero`), triggered when a featured template changes meaningfully.

Dependencies:

- `playwright` (devDep). Installs a chromium binary (~150MB) on first install. Acceptable: contributors only pay this if they run `pnpm hero`. Running `pnpm install` does NOT auto-install browsers (Playwright needs `pnpm exec playwright install chromium` separately, which we add as a comment in the script header). For the maintainer's first run we'll do `pnpm exec playwright install chromium`.
- `sharp` (devDep). Native binary, prebuilt for darwin-arm64 and linux-x64. Standard imaging library.

## Risks

- **Hero PNG drift.** If a featured template's visuals change and someone forgets to re-run `pnpm hero`, the README stops reflecting the live product. Mitigation: a TODO comment at the top of `build-hero.mjs` reminds future maintainers to refresh after meaningful template changes. Acceptable: a stale hero is still better than no hero.
- **Hero PNG size.** A 1280×640 PNG of mostly text-on-white compresses well, target under 400KB. If it overshoots, sharp gets a `palette: true` flag to drop to 8-bit indexed color.
- **Playwright chromium download** on first hero run is slow (~150MB) but only paid by maintainers running the script.
- **CSP gating.** `resumex.karnstack.com/preview/*` already allows iframing from karnstack and localhost (verified in `public/_headers`), but the script doesn't iframe. It loads the URL directly in a Playwright page, so there are no CSP issues.
